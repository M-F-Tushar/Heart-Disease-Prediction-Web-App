## How to Contribute

### Reporting Issues
- Check if the issue already exists in the [Issues](https://github.com/M-F-Tushar/Heart-Disease-Prediction-Web-App/issues) section
- Provide a clear description of the problem
- Include steps to reproduce the issue
- Add screenshots if applicable

### Suggesting Enhancements
- Open an issue describing your enhancement idea
- Explain why this enhancement would be useful
- Provide examples if possible

### Pull Requests
1. **Fork the repository** and create your branch from `main`
2. **Make your changes** with clear, descriptive commits
3. **Test your changes** to ensure the app still works correctly
4. **Update documentation** if you've made changes that affect it
5. **Submit a pull request** with a clear description of your changes

## Development Setup

1. Clone your fork:
   ```bash
   git clone https://github.com/YOUR_USERNAME/Heart-Disease-Prediction-Web-App.git
   cd Heart-Disease-Prediction-Web-App
   ```

2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```

3. Run the app locally:
   ```bash
   streamlit run app.py
   ```

## Code Style
- Follow PEP 8 guidelines for Python code
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions small and focused

## Testing
- Test your changes thoroughly before submitting
- Ensure the Streamlit app runs without errors
- Verify predictions work correctly

## Questions?
Feel free to open an issue for any questions about contributing!
